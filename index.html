PEP 8 — Código que se puede leer

Teoría

PEP 8 es la guía oficial de estilo para Python. No es obligatoria, pero todo el mundo profesional la usa. Aplicarla hace que tu código sea más legible y fácil de mantener.

Regla❌ Incorrecto✅ Correcto
Nombres de variablesprecioTotal, PrecioTotalprecio_total
Constantesiva = 0.19IVA = 0.19
Espacios alrededor de =x=5+3x = 5 + 3
Líneas en blancoTodo pegado sin separarSección entrada / proceso / salida separadas
Líneas largasUna línea de 150+ caracteresMáximo ~79 caracteres por línea
Comentarios#esto es malo# Esto es correcto (espacio)
ComillasMezclar " y 'Elegir uno y ser consistente
💡

Atajo en VSCode: La extensión Pylance subraya en amarillo cuando rompes PEP 8. También puedes usar Shift+Alt+F (Win) / Shift+Option+F (Mac) para formatear automáticamente.

🧹

PEP 8 — Antes y después

Ejemplo
❌ Sin PEP 8 — difícil de leer
n=input("nombre:")
e=int(input("edad:"))
p=float(input("peso:"))
a=float(input("altura:"))
imc=p/a**2
print(f"hola {n} tu imc es {imc}")
✅ Con PEP 8 — limpio y profesional
# ── ENTRADA ──
nombre  = input("Nombre: ")
edad    = int(input("Edad: "))
peso    = float(input("Peso (kg): "))
altura  = float(input("Altura (m): "))

# ── PROCESO ──
imc = peso / altura ** 2

# ── SALIDA ──
print(f"Hola {nombre}, tu IMC es {imc:.2f}")
🐍

El código se lee muchas más veces de las que se escribe. Un código limpio te ahorra horas de trabajo en el futuro. Adopta PEP 8 desde el primer día.

Los 5 errores más frecuentes

Debugging

Aprende a leer el mensaje de error. Python siempre te dice qué falló y en qué línea. No le temas al error rojo — es tu mejor profesor.

🔴 Error 1: TypeError — str + int
edad = input("Edad: ")
print("En 10 años tendrás " + (edad + 10) + " años")
# TypeError: can only concatenate str (not "int") to str
✅ Solución
edad = int(input("Edad: "))  # convertir al leer
print(f"En 10 años tendrás {edad + 10} años")
🔴 Error 2: NameError — variable sin definir
print(resultado)
# NameError: name 'resultado' is not defined
✅ Solución
resultado = 5 * 3  # definir ANTES de usar
print(resultado)
🐛

Los 5 errores más frecuentes (cont.)

Debugging
🔴 Error 3: ValueError — input no convertible
edad = int(input("Edad: "))
# Usuario escribe "veinte" → ValueError: invalid literal for int()
✅ Solución
# Guiar al usuario con el mensaje
edad = int(input("Edad (ej: 22): "))
🔴 Error 4: ZeroDivisionError
divisor = int(input("Divisor: "))
res = 10 / divisor   # si usuario escribe 0 → Error!
✅ Solución (tema de Sesión 5 — por ahora avisa en el mensaje)
divisor = int(input("Divisor (distinto de 0): "))
🔴 Error 5: SyntaxError — f-string sin la f
print("Hola {nombre}, tienes {edad} años")
# Imprime literalmente: Hola {nombre}, tienes {edad} años
✅ Solución
print(f"Hola {nombre}, tienes {edad} años")
# La f antes de las comillas activa las llaves {}

# ── MENÚ ──
print("╔══════════════════════════╗")
print("║    🧮 CALCULADORA SENA   ║")
print("╠══════════════════════════╣")
print("║  1 → Suma                ║")
print("║  2 → Resta               ║")
print("║  3 → Multiplicación      ║")
print("║  4 → División            ║")
print("║  5 → Potencia            ║")
print("║  6 → Módulo (residuo)    ║")
print("╚══════════════════════════╝")

# ── ENTRADA ──
opcion = int(input("\nElige operación (1-6): "))

print(f"  Multiplic : {multi:>10,.4f}")
print(f"  División  : {division:>10,.4f}")
print(f"  Potencia  : {potencia:>10,.4f}")
print(f"  Módulo    : {modulo:>10,.4f}")
print(f"{'─'*28}")

Calculadora V2 — Con resultado seleccionado

Versión 2

V2 mejora la salida: en lugar de mostrar todo, muestra solo el resultado de la operación elegida.

Python · Calculadora V2 — reemplaza la sección SALIDA 
# ── SALIDA V2 ── (reemplaza la sección SALIDA de V1)
operaciones = {
    1: ("Suma",           num1 + num2, "+"),
    2: ("Resta",          num1 - num2, "-"),
    3: ("Multiplicación", num1 * num2, "×"),
    4: ("División",       num1 / num2, "÷"),
    5: ("Potencia",       num1 ** num2, "^"),
    6: ("Módulo",         num1 %  num2, "%"),
}

nombre_op, resultado, simbolo = operaciones[opcion]

print(f"\n{'='*34}")
print(f"  RESULTADO · {nombre_op.upper()}")
print(f"{'='*34}")
print(f"  {num1:>8,.4f}  {simbolo}")
print(f"  {num2:>8,.4f}")
print(f"  {'─'*16}")
print(f"  {resultado:>8,.4f}  ← resultado")
print(f"{'='*34}")
